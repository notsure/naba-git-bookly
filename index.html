Automatically add some description to your pull request

If you are using Github to manage your project, either for business or private, you all already had to make a pull request. If you don't know what github or a pull request is, leave this page, this is none of your business!

What does the script do?

When ever I have to make a pull request for work it has to follow some styling guidelines. For example we started to always add a Task and an Information sector in the pull requests description. This helps to remember what needs to be done and helps the reviewer to understand what the pull request is about.

A proper pull request would look like this

However this is how a pull request usually looks like

As you can see it looks pretty shitty and a you are giving the reviewer a hard time by not formatting your pull request. If there is only one single pull request in your lifetime you probably do not need to change anything, as you will love to describe the reviewer what you've done there. However for those of you who have to create dozens of pull requests every day it is pretty annoying to format the pull requests every single time.

The Solution

The copy 'n' past time is over! Add this bookmark to your brower to automatically add the base structure to your pull request. All you need to do is change the tasks you have done. Save time, conserve your keyboard.

Bookmark me!

How it Works

Make the changes you need in your project. Publish and push your branch. Visit Github and click on your branch and make a pull request. Now click the bookmark. Thats it.
